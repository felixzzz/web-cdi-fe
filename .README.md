# Corporate Website Documentation

Dokumentasi teknis untuk pengembangan website company profile. Project ini dibangun menggunakan **Next.js (App Router)** dengan arsitektur berbasis fitur (*feature-based architecture*) dan dukungan multi-bahasa (**i18n**).

## 🛠 Tech Stack

- **Framework:** Next.js 14+ (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS & Shadcn UI (`components/ui`)
- **Validation:** Zod
- **I18n:** Routing berbasis Locale (`[locale]`)

## Struktur Project

Memahami struktur folder adalah kunci untuk berkontribusi di project ini:

### 1. App Routing (`app/`)
- **`[locale]/`**: Root folder untuk semua halaman publik. Routing dinamis untuk bahasa (ID/EN).
- **`(sitemaps)/`**: Endpoint khusus untuk generate XML sitemap per modul.
- **`api/`**: Server-side logic (jika ada).

### 2. Components (`components/`)
Kami memisahkan komponen menjadi 3 level:
- **`ui/`**: *Atomic components* (Button, Input, Card). Reusable & Generic. **Jangan ubah logic di sini.**
- **`shared/`**: Komponen global (Navbar, Footer, ProgressBar, CookieConsent).
- **`features/`**: Komponen spesifik per halaman/modul.
  - *Contoh:* Folder `features/Sustainability/` hanya berisi komponen untuk halaman Sustainability.

### 3. Data & Logic Layer
- **`services/`**: Layer untuk fetch data/API call. **Wajib** gunakan service, jangan fetch langsung di component.
  - *Pattern:* `services/[Module]/[Feature]Service.ts`
- **`types/`**: Definisi Interface/Type TypeScript yang mencerminkan respons API.
- **`dictionaries/`**: JSON file (`en.json`, `id.json`) untuk translasi static text.
- **`schemas/`**: Skema validasi form (Zod).
